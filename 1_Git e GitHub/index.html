Git E GitHub

Para começarmos a falar sobreesses dois elementos, vamos falar primeiro sobre Repositórios Locais e Remotos. Reositórios são locais onde podemos criar cópias de segurança dos nossos códigos e voltar sempre que necessário e isso pode ser feito tanto no próprio computador como em um servidor remoto.

É muito comum que ao programar façamos alguma alteração no nosso site e desejamos ir para uma alteraçao anterior por qcharmos que ficar+a melhor, mas isso se torma muito trabalhoso porque para isso, teriamos que guardar diversas cópias do código. Para resolver essa situação foi criado o GIT. Essa ferramenta permite fazer backups dos códigos baseando-se em versões. Durante a criação do site, vamos criando algumas versões, ou seja, definimos em certo mometno da criação uma versão e poderemos voltar fácilmete caso seja necessário. A isso chamamos de Versionamento e isso funciona com qualquer linguagem. No entanto, caso aconteça algo com o computador e isso afecte o HD, os dados sejam perdidos, todas as versões do documento serão perdidas e Nada feito. Outra situação: Caso quisermo mostrar um site para alguém, podemos usar o GitHub para hospedar o site remotamente e qualquer pessoa poderá acessar. O fixe é que pode se criar equipas, assim, qualquer pessoa autorizada poderá fazer alterações no site
